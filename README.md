# 智慧物业知识库咨询平台

## 知识库操作手册

智慧物业知识库咨询平台是一套面向物业服务场景的 Web 系统，包含用户端咨询入口、管理员知识库维护后台、咨询日志分析、用户反馈闭环和 RAG 向量检索问答能力。普通用户可以通过在线知识库和智能问答快速获取物业缴费、报修、停车、装修、生活服务、投诉建议等常见问题答案；管理员可以维护标准问答、分类、反馈、咨询日志和向量库，让系统回答持续变准。

本文档用于介绍项目结构、运行方式和网页功能操作流程。首次使用建议先阅读“快速启动”和“初始化数据”，再按“用户端操作手册”或“管理员端操作手册”进行业务操作。

## 项目功能概览

### 用户端功能

- 登录和注册：支持普通用户通过账号或邮箱登录，新用户可通过邮箱注册。
- 用户首页：展示平台入口、数据概览、热门问题和快捷操作。
- 在线知识库：支持关键词搜索、分类筛选、查看知识详情、复制答案和提交有用性反馈。
- 智能问答：用户输入自然语言问题，系统基于 FAISS 向量库召回知识片段，并调用 DeepSeek 生成回答；未命中时提示联系人工客服。
- 我的咨询记录：查看个人历史提问、回答、命中状态、分类和是否需要人工处理。
- 我的反馈：查看自己提交的反馈和处理状态。

### 管理端功能

- 工作台：查看知识库总数、咨询量、反馈数量、向量库状态和快捷入口。
- 知识库维护：新增、编辑、删除、发布、停用标准问答；支持 CSV 批量导入和重建向量库。
- 分类管理：维护知识分类名称、描述、排序和启用状态。
- 反馈处理：查看用户反馈，标记处理、忽略反馈，或将有效反馈补充为知识库问答。
- 咨询日志：检索用户咨询记录，筛选未命中问题，并将问题沉淀到知识库。
- 数据看板：通过图表查看咨询趋势、分类分布、命中率和反馈状态。
- 向量库维护：查看 RAG 向量库状态，基于已发布知识重建 FAISS 索引。

### RAG 问答能力

系统后端使用 `BAAI/bge-small-zh-v1.5` 作为中文 embedding 模型，将已发布知识转为向量并写入本地 FAISS 索引。用户提问时，系统先检索最相近的知识片段，再构造提示词调用 DeepSeek；如果检索分数低于阈值或模型调用失败，会回退到知识库标准答案或未命中提示。

为避免用户第一次登录后首次提问时因 Hugging Face 模型下载或加载导致超时，后端已将模型缓存固定到本地目录。

## 快速启动

### 1. 克隆项目

```bash
git clone https://github.com/ddlllooo/Propertyknowledgebase.git
cd Propertyknowledgebase
```

### 2. 安装前端依赖

```bash
npm install
```

### 3. 安装后端依赖

进入后端目录并创建虚拟环境：

```bash
cd property_kb_backend
python -m venv .venv
```

Windows PowerShell：

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

macOS 或 Linux：

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

### 4. 配置后端环境变量

在 `property_kb_backend/` 下创建 `.env` 文件。不要把真实密钥提交到 Git。

```env
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

EMBEDDING_MODEL_NAME=BAAI/bge-small-zh-v1.5
FAISS_INDEX_DIR=storage/faiss_index
HF_CACHE_DIR=storage/hf_cache

RAG_TOP_K=3
RAG_MAX_DISTANCE_THRESHOLD=1.2
RAG_PRELOAD_EMBEDDINGS=true
```

说明：

- `HF_CACHE_DIR` 用于固定 Hugging Face 模型缓存，避免每次启动重新下载。
- `RAG_PRELOAD_EMBEDDINGS=true` 表示后端启动时预热 embedding 模型，减少用户第一次提问时的等待。
- 如果临时不希望启动时加载 embedding 模型，可设置 `RAG_PRELOAD_EMBEDDINGS=false`。

### 5. 配置 MySQL

默认数据库连接在 `property_kb_backend/config.py` 中：

```python
mysql+pymysql://your mysqlname:password@localhost:port(3306)/property_kb?charset=utf8mb4
```

请先在 MySQL 中创建数据库：

```sql
CREATE DATABASE property_kb DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

如需修改数据库账号、密码或地址，可调整 `property_kb_backend/config.py` 中的 `SQLALCHEMY_DATABASE_URI`。

### 6. 初始化数据库表和演示数据

在 `property_kb_backend/` 目录执行：

```bash
python init_db.py
python seed_data.py
```

初始化后默认账号如下：

| 角色 | 账号 | 邮箱 | 密码 |
| --- | --- | --- | --- |
| 普通用户 | `user` | `user@example.com` | `123456` |
| 管理员 | `admin` | `admin@example.com` | `123456` |

### 7. 启动后端

在 `property_kb_backend/` 目录执行：

```bash
python app.py
```

默认服务地址：

```text
http://localhost:5000
```

健康检查：

```text
GET http://localhost:5000/api/ping
```

### 8. 启动前端

回到项目根目录执行：

```bash
npm run dev
```

默认前端地址：

```text
http://localhost:5173
```

### 9. 生产构建

```bash
npm run build
```

构建产物输出到 `dist/`，该目录已加入 `.gitignore`。

## 首次使用流程

管理员首次配置建议按以下顺序操作：

1. 初始化数据库表和演示数据。
2. 登录管理员账号 `admin / 123456`。
3. 进入“分类管理”，确认分类启用状态。
4. 进入“知识库维护”，确认需要参与问答的知识状态为“已发布”。
5. 进入“向量库维护”，点击“重建向量库”。
6. 等待向量库构建完成后，使用普通用户账号进入“智能问答”测试。
7. 如果回答未命中，管理员可在“咨询日志”或“反馈处理”中将问题补充到知识库，再重建向量库。

重要提醒：RAG 检索只使用已发布知识。草稿、待审核、已停用的问答不会进入向量检索结果。

## 用户端操作手册

### 1. 登录

访问 `http://localhost:5173/login`。

操作步骤：

1. 输入账号或邮箱。
2. 输入密码。
3. 点击“登录”。
4. 系统会根据账号角色自动进入用户首页或管理端首页。

普通用户演示账号：

```text
账号：user
密码：123456
```

### 2. 注册

在登录页点击注册入口。

填写要求：

- 邮箱必须是有效邮箱格式。
- 密码长度不少于 6 位。
- 密码必须同时包含字母和数字。
- 两次输入密码必须一致。

注册成功后，新账号默认为普通用户角色。

### 3. 用户首页

用户登录后进入“用户首页”。该页面提供：

- 平台能力概览。
- 常用功能入口。
- 推荐知识分类。
- 热门问题或快捷查询入口。

常见操作：

- 点击“查询知识库”进入在线知识库。
- 点击“智能问答”进入 AI 问答页面。
- 点击“我的咨询记录”查看历史提问。

### 4. 在线知识库

入口：

```text
用户侧边栏 -> 在线知识库
```

页面用途：按关键词或分类查找标准问答。

操作步骤：

1. 在搜索框输入关键词，例如“物业费”“漏水”“停车”“装修”。
2. 点击分类按钮筛选对应分类。
3. 点击知识卡片打开知识详情。
4. 在详情抽屉中查看标准问题、标准答案、分类和关键词。
5. 点击“复制答案”可复制标准答案。
6. 点击“有帮助”或“没帮助”提交知识反馈。

说明：

- 搜索会自动触发，不需要手动刷新。
- “有帮助”和“没帮助”会进入反馈处理流程，管理员可在后台查看。

### 5. 智能问答

入口：

```text
用户侧边栏 -> 智能问答
```

页面用途：用自然语言向智慧物业助手提问。

操作步骤：

1. 在底部输入框输入问题。
2. 点击“发送”，或按 Enter 发送。
3. 如需换行，按 Shift + Enter。
4. 等待系统返回回答。
5. 查看回答下方的分类、来源、参考问题和相似度。
6. 对回答点击“有帮助”“没帮助”或“需要人工处理”提交反馈。

示例问题：

- 物业费可以通过哪些方式缴纳？
- 家中漏水应该如何报修？
- 临时车辆如何进入小区？
- 装修施工时间有什么规定？

回答状态说明：

- 已命中：系统在知识库中找到了相似问题，并生成或返回回答。
- 未命中：当前知识库没有足够明确的答案，建议联系人工客服。
- 需要人工处理：用户认为系统回答无法解决问题，需要管理员进一步处理。

### 6. 我的咨询记录

入口：

```text
用户侧边栏 -> 我的咨询记录
```

页面用途：查看当前账号提交过的所有智能问答记录。

可查看内容：

- 咨询时间
- 用户问题
- 系统回答
- 命中分类
- 相似度
- 命中状态
- 是否需要人工处理

使用建议：

- 如果某次回答不准确，可回到智能问答页面对该回答提交“没帮助”或“需要人工处理”反馈。
- 对未命中的问题，管理员会通过咨询日志或反馈处理进行知识库补充。

### 7. 我的反馈

入口：

```text
用户侧边栏 -> 我的反馈
```

页面用途：查看自己提交的反馈及管理员处理状态。

反馈状态一般包括：

- 待处理：管理员尚未处理。
- 处理中：管理员正在核实。
- 已处理：管理员已处理并填写说明。
- 已忽略：管理员判断暂不调整知识库。

## 管理员端操作手册

管理员演示账号：

```text
账号：admin
密码：123456
```

管理员登录后会进入管理端，左侧菜单包含工作台、知识库维护、分类管理、反馈处理、咨询日志、数据看板和向量库维护。

### 1. 工作台

入口：

```text
管理端侧边栏 -> 工作台
```

页面用途：查看系统运行概况和快捷操作。

可查看内容：

- 知识库总数
- 咨询量
- 反馈数量
- 向量库状态
- 常用快捷入口

推荐日常流程：

1. 查看是否有待处理反馈。
2. 查看是否有未命中咨询。
3. 进入知识库维护补充问答。
4. 补充后重建向量库。

### 2. 知识库维护

入口：

```text
管理端侧边栏 -> 知识库维护
```

页面用途：维护标准问答，是系统问答质量的核心数据来源。

#### 新增问答

操作步骤：

1. 点击“新增问答”。
2. 填写“标准问题”。
3. 填写“标准答案”。
4. 选择或输入分类。
5. 设置状态，建议可直接设为“已发布”。
6. 添加关键词。
7. 填写数据来源，例如“物业服务合同”“工程报修服务规范”。
8. 点击“保存”。
9. 如需让智能问答立即使用新增内容，点击“重建向量库”。

字段说明：

- 标准问题：用户可能提出的问题表达。
- 标准答案：面向用户展示的正式答复。
- 分类：用于检索、筛选和统计。
- 关键词：帮助管理员管理知识，也便于搜索。
- 数据来源：用于追溯答案依据。
- 状态：只有“已发布”的知识会进入 RAG 向量库。

#### 编辑问答

操作步骤：

1. 在表格中找到目标问答。
2. 点击“编辑”。
3. 修改问题、答案、分类、关键词、来源或状态。
4. 点击“保存”。
5. 如果修改了会影响智能问答的内容，重建向量库。

#### 发布或停用问答

操作步骤：

1. 在表格操作列点击“发布”或“停用”。
2. 确认状态变化。
3. 重建向量库，使向量检索同步最新发布状态。

说明：

- 停用后的问答不会再作为用户知识库和 RAG 检索结果。
- 草稿、待审核内容适合暂存，不建议直接用于用户问答。

#### 删除问答

操作步骤：

1. 点击目标问答操作列的“删除”。
2. 确认删除。
3. 重建向量库。

注意：删除不可逆，建议仅删除明显错误或重复数据。

#### 批量导入问答

操作步骤：

1. 点击“批量导入”。
2. 点击“下载导入模板”。
3. 按模板填写 CSV 文件。
4. 点击“选择 CSV 文件”，或拖拽文件到导入区域。
5. 检查页面预览和错误提示。
6. 点击“导入 N 条”。
7. 导入完成后重建向量库。

CSV 字段：

```text
question,answer,category,keywords,source,status
```

填写规则：

- `question`：必填，标准问题。
- `answer`：必填，标准答案。
- `category`：必填，分类名称。
- `keywords`：建议填写多个关键词，可用中文逗号、英文逗号或分号分隔。
- `source`：建议填写制度、合同、规范或来源说明。
- `status`：建议填写“已发布”“草稿”“待审核”或“已停用”。

### 3. 分类管理

入口：

```text
管理端侧边栏 -> 分类管理
```

页面用途：维护知识分类结构。

操作步骤：

1. 点击“新增分类”。
2. 填写分类名称。
3. 填写分类描述。
4. 设置排序值，数值越小越靠前。
5. 设置状态为启用或停用。
6. 点击保存。

管理建议：

- 分类名称应简洁明确，例如“物业缴费”“报修服务”“停车管理”。
- 不建议频繁修改已大量关联问答的分类名称。
- 停用分类前，请先确认该分类下是否仍有已发布知识。

### 4. 反馈处理

入口：

```text
管理端侧边栏 -> 反馈处理
```

页面用途：处理用户对回答质量的反馈。

可筛选条件：

- 反馈状态
- 反馈类型
- 关键词

常见处理方式：

#### 查看详情

点击“查看详情”，可查看：

- 用户问题
- AI 回答
- 命中分类
- 相似度
- 反馈类型
- 用户建议
- 管理员处理说明

#### 标记已处理

操作步骤：

1. 点击“标记已处理”。
2. 填写管理员处理说明。
3. 点击确认。

适用场景：

- 已电话联系用户。
- 已核实该回答无误。
- 已在其他系统派单处理。

#### 加入知识库

操作步骤：

1. 点击“加入知识库”。
2. 确认或修改标准问题。
3. 填写更准确的标准答案。
4. 选择分类。
5. 添加关键词。
6. 填写来源。
7. 保存到知识库。
8. 到“知识库维护”检查内容并重建向量库。

适用场景：

- 用户问题未命中。
- 系统回答不准确。
- 用户建议中包含可沉淀的新知识。

#### 忽略反馈

适用场景：

- 重复反馈。
- 用户误点。
- 反馈内容与知识库无关。

忽略后建议填写处理说明，方便后续追踪。

### 5. 咨询日志

入口：

```text
管理端侧边栏 -> 咨询日志
```

页面用途：查看所有用户智能问答记录，分析知识库覆盖情况。

可筛选条件：

- 关键词
- 分类
- 命中状态
- 是否需要人工
- 日期范围

重点功能：

#### 仅查看未命中问题

点击“仅查看未命中问题”，快速定位当前知识库覆盖不足的问题。

建议处理流程：

1. 查看未命中问题。
2. 点击“查看详情”确认用户原问题和系统回答。
3. 点击“加入知识库”。
4. 填写标准问题、答案、分类、关键词和来源。
5. 保存后重建向量库。
6. 使用普通用户账号重新提问验证效果。

#### 加入知识库

咨询日志中的“加入知识库”适合将真实用户问题转为标准问答。补充时建议将口语化问题整理为更标准的问题标题，并给出明确、可执行的答案。

### 6. 数据看板

入口：

```text
管理端侧边栏 -> 数据看板
```

页面用途：通过数据分析评估知识库运营效果。

主要指标：

- 咨询趋势
- 分类占比
- 命中率
- 未命中数量
- 反馈状态分布
- 高频分类和问题

使用建议：

- 命中率下降时，优先查看咨询日志中的未命中问题。
- 某分类咨询量高但反馈差时，优先检查该分类下的标准答案质量。
- 反馈待处理数量过多时，应及时进入反馈处理页面闭环。

### 7. 向量库维护

入口：

```text
管理端侧边栏 -> 向量库维护
```

页面用途：管理 RAG 问答依赖的 FAISS 向量索引。

可查看内容：

- 向量库状态
- 知识数量
- 最近构建时间
- 向量片段数量
- Embedding 模型
- LLM 状态
- 同步结果

#### 重建向量库

操作步骤：

1. 确认知识库中有“已发布”问答。
2. 点击“重建向量库”。
3. 等待构建完成。
4. 使用智能问答测试新知识是否生效。

触发重建的常见场景：

- 新增问答后。
- 修改已发布问答的标准答案后。
- 发布或停用问答后。
- 从反馈或咨询日志补充知识后。
- 批量导入问答后。

注意事项：

- 如果没有已发布知识，重建会失败。
- 首次构建会加载 embedding 模型，可能比后续构建更慢。
- 模型缓存保存在 `property_kb_backend/storage/hf_cache/`，FAISS 索引保存在 `property_kb_backend/storage/faiss_index/`。

## 知识库运营建议

### 标准问答编写原则

- 问题要贴近用户真实表达，但标题应尽量标准。
- 答案要明确、简洁、可执行。
- 不要编造费用、电话、政策、时间等敏感信息。
- 涉及收费、合同、装修、停车等规则时，应填写来源。
- 一个问答只解决一个主要问题，避免答案过长或覆盖过宽。

### 分类维护建议

- 分类数量不宜过多，建议 5 到 12 个。
- 分类名称保持稳定，避免频繁更名。
- 新分类上线前先补充足够问答，否则用户筛选后可能为空。

### 反馈闭环建议

- 每天查看“反馈处理”和“咨询日志”。
- 优先处理未命中、需要人工、没帮助的记录。
- 将高频问题补充到知识库后及时重建向量库。
- 对已处理反馈填写说明，便于后续追踪。

### 向量库维护建议

- 批量修改知识后统一重建向量库。
- 如果用户反馈“新知识没有生效”，先检查问答是否为“已发布”，再检查是否已重建向量库。
- 如果首次问答慢，检查 Hugging Face 模型缓存是否已经下载完成。


## 常见问题

### 1. 用户第一次提问超时怎么办？

原因通常是后端首次加载或下载 embedding 模型。当前项目已将 Hugging Face 缓存固定到 `property_kb_backend/storage/hf_cache/`，并在后端启动时预热模型。

处理建议：

1. 保持 `RAG_PRELOAD_EMBEDDINGS=true`。
2. 第一次启动后端时等待模型下载完成。
3. 后续启动会复用缓存。
4. 如果 Hugging Face 下载限速，可配置 `HF_TOKEN` 提高稳定性。

### 2. 后端提示未配置 DeepSeek API Key 怎么办？

检查 `property_kb_backend/.env` 是否包含：

```env
DEEPSEEK_API_KEY=你的 DeepSeek API Key
```

如果没有配置，RAG 检索仍可能命中知识库，但 DeepSeek 生成会失败，系统会回退到知识库标准答案或未命中提示。

### 3. 新增知识后智能问答没有命中怎么办？

请依次检查：

1. 问答状态是否为“已发布”。
2. 是否点击了“重建向量库”。
3. 用户提问是否与标准问题或答案语义相关。
4. `RAG_MAX_DISTANCE_THRESHOLD` 是否过低。
5. FAISS 索引目录是否存在 `index.faiss`。

### 4. 向量库重建失败怎么办？

常见原因：

- 没有已发布知识。
- MySQL 数据库连接失败。
- embedding 模型下载失败。
- 本地 `storage` 目录没有写入权限。

处理建议：

1. 进入知识库维护，确认至少一条问答为“已发布”。
2. 检查 MySQL 服务是否启动。
3. 检查网络是否可以访问 Hugging Face。
4. 删除损坏的 `storage/faiss_index/` 后重新构建。

### 5. 前端请求提示未登录或无权限怎么办？

可能原因：

- JWT 已过期。
- 当前账号角色不匹配。
- 浏览器 `sessionStorage` 中 token 异常。

处理方法：

1. 退出登录后重新登录。
2. 普通用户访问 `/user/*` 页面。
3. 管理员访问 `/admin/*` 页面。
4. 如仍异常，清空浏览器站点数据后重试。

## 日常维护流程

### 每日运营建议

1. 登录管理员账号。
2. 查看工作台是否存在待处理反馈。
3. 进入咨询日志筛选未命中问题。
4. 将高频问题沉淀到知识库。
5. 重建向量库。
6. 使用普通用户账号抽样测试问答效果。

### 知识更新发布流程

1. 新增或编辑问答。
2. 设置状态为“已发布”。
3. 检查分类、关键词和来源。
4. 点击“重建向量库”。
5. 到智能问答页面验证。
6. 观察反馈和咨询日志。

### 故障排查流程

1. 先访问 `/api/ping` 确认后端是否在线。
2. 检查浏览器控制台和 Network 请求。
3. 检查后端终端日志。
4. 检查 MySQL 连接。
5. 检查 `.env` 中 DeepSeek 和 RAG 配置。
6. 检查向量库是否已构建。
